Home

The Home section is the default view displayed when OneVision starts. It is the primary interface for production, where you can monitor and interact with the machine vision system. This section is designed to provide a comprehensive overview of the production process, enabling you to manually control cameras, execute AI model inferences, and view results in real-time.

Layout​

The Home section consists of the following main areas:

  • Action Bar
  • Production view
  • 2D Display view

Action Bar​

The Action Bar provides quick access to essential functions for managing the machine vision system within OneVision. These buttons facilitate various operations such as starting or stopping program execution, controlling cameras, executing AI model inferences, and handling image operations.

  • Start/Stop Program: Initiates or halts the execution of sequences. This button toggles the program's runtime, allowing you to start or stop the flow of operations.
  • Single Shot: Triggers a single image capture from the configured camera.
  • Start Video: Begins streaming video from the camera.
  • Start Acquisition: Initiates continuous image acquisition from the camera.
  • Software Trigger: Sends a software trigger to the camera if supported. This action is available when the camera is actively acquiring images.
  • Switch Camera: Allows switching between different configured cameras. This action is available when multiple cameras are configured.
  • Inference: Executes an AI model inference on the current image. This action is available when a brain is loaded.
  • Switch Brain: Enables selection of different AI models (brains) for inference. This action is available when multiple AI models are loaded.
  • Open Image: Opens an image from disk for analysis.
  • Save Image: Saves the current image displayed in the workspace to disk.
info

Except for the Stop Program action, which halts the runtime, the other actions are designed to be used when the runtime is stopped. During production, these actions are disabled to prevent interference with ongoing operations.

Production View​

The Production View in OneVision serves as the central area where configurable elements are displayed and interacted with. This view is highly customizable, allowing users to set up the layout that best fits their specific application needs. It enables users to monitor and manage various aspects of the machine vision system during production.

Topbar​

The Topbar within the Production View is used to display global information related to the production process. Users can customize the color of the topbar, the text displayed in the center, and various properties inside it.

The Topbar acts as a dropdown menu if at least one property is added, allowing users to click on it to view the properties. If no properties are added, the topbar cannot be opened. It can also show additional hidden information such as image sizes or pixel formats when in 2D display mode. During runtime, it may display real-time metrics like cycle execution time or counters.

tip

In some applications, it is useful to change the topbar color based on inspection results to alert workers on the production line.

Visu Elements​

The Visu Elements are the configurable components within the Production View workspace. These elements allow users to customize their interface according to the specific needs of their application. The following types of Visu Elements are available:

  • Display: For displaying and changing images.
  • Time Graph: For visualizing data over time.
  • Text Box: For displaying text.
  • Static Image: For displaying static images, such as logos.
  • Table: For displaying tabular data.
  • Push Button: For initiating actions.
  • Toggle Button: For toggling between states.
  • Text Input: For user input.
  • Selector: For selecting options from a list.

These elements can be added and configured in the Visualization section. Additionally, in the Programming section, users can utilize the API of each element to enable interaction and response within the program workflow.

For example, consider the following scenario:

  1. An image is loaded from disk.
  2. An inference is made (with the AI brain loaded) on the image to detect cookies and chocolate chips.
  3. The image, with detected cookies, is sent to the main display.
  4. The results are processed, filtering cookies above a certain threshold.
  5. The selector options are filled with the filtered cookies.
  6. The system waits for a user action: either to select a cookie from the selector or to click the next image button.
  7. If a user selects a cookie, the image is cropped to show only the selected cookie, which is then displayed in a smaller display along with its chocolate chips.
  8. The table data below the smaller display is updated with the index of the cookie and the number of chocolate chips.
  9. If the user clicks on the next image button, the next image in the folder is loaded, and the process repeats.

This example illustrates the vast possibilities that the combination of Visu Elements offers, enabling highly customizable and interactive production workflows.

info

Check the API of each visu element type in the Programming reference

Double clicking on an image display you enter to the 2D Display Mode

Side Panel​

The Side Panel is a collapsible container that houses tabs, each offering specific functionality depending on whether you are in the Production View or the 2D Display View.

Actions​

Within the Production View, the Side Panel only includes the Actions tab. This tab provides a suite of tools designed to streamline and expand the interaction capabilities within the Production View.

info

Examples of actions include a counter to track production quantities, a serial number identifier for production batches, or a toggle button to activate a flag for saving images. For detailed guidance on using these actions effectively, refer to the Actions section.

2D Display Mode​

The 2D Display mode in OneVision provides an advanced interface for detailed image analysis and manipulation. It consists of the following key areas:

You can enter the 2D Display mode in two ways:

  • Double-Click on a Display Element: In the Production View, double-click on a display element to open it in 2D Display mode.
  • Manual Camera Operation: Perform a manual camera operation such as a single shot, start video, start acquisition, or open image. This will open the Manual Display in 2D Display mode.

Topbar​

The Topbar in the 2D Display mode displays global information related to the current image analysis session. It allows users to customize its appearance by adjusting colors, modifying displayed text, and adding additional properties. When properties are added, the Topbar functions as a dropdown menu, enabling access to specific details related to the analysis.

  • Display Selector: Located on the left side, it allows switching between different configured displays and the Manual Display where images loaded from disk or video streams are sent for analysis.
  • Capture Result: The main text displayed in the middle of the topbar.
  • Topbar Properties: As with the topbar in the Production View, global data of the capture that can be added dynamically.
  • Pin Button: Located next to the close button, this is used to pin the current capture. This is useful when continuous captures are being sent, but the user wants to focus on the current one. Pinning it will prevent the display from updating. See more in the History Tab of the Side Panel.
  • Close Button: Located on the right side, it allows users to exit the 2D Display mode and return to the Production View.

Image Display​

The Image Display serves as the central workspace where images or video frames are presented for detailed analysis. Users can interact with the displayed content in the following ways:

  • Zoom: Adjust the magnification level using Ctrl + mouse scroll or other zoom tools.
  • Pan: Navigate within the image by dragging with the mouse.
  • Interactive Vision Elements: Click on elements like rectangles, polygons, points, and lines painted over the image to mark visual areas, which are results of the post-processing done on the image.

Display Tools​

The Display Tools are located in a bottom bar below the Image Display area, containing essential tools designed for image analysis and navigation:

  • Zoom In/Out: Adjust the zoom level incrementally.
  • Set to 1:1: Reset the image display to its original size.
  • Zoom Percentage: Displays the current zoom level for reference.
  • Pointer Position: Indicates the coordinates of the cursor within the image.
  • Pixel Value: Provides detailed information about the pixel under the cursor, such as color value or intensity.
  • Display Configuration: Customize display settings, including color mapping and overlay options.
  • Capture Selector: Select and capture specific image areas or regions for further analysis or documentation.

Functionality​

  • Detailed Analysis: Perform precise measurements, annotations, and inspections directly on displayed images or video frames.
  • Interactive Tools: Access tools for annotation, measurement, and interactive operations within the Image Display area.
  • Integration: Seamlessly integrates with the overall production workflow, facilitating quick transitions between overview monitoring (Production View) and detailed analysis (2D Display mode).
