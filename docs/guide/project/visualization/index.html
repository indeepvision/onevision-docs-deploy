Visualization

The Visualization section lets the user to configure the elements that can be displayed in the Home page [See here]. These elements are accessible from the Programming [See here], to update its properties during production. By default, the table is empty.

The main goal of visual elements is to enable user interaction in production. This can be achieved by displaying information from processed results (images, charts, text, numbers) or by allowing the user to input some type of information (press a button, select an item from a dropdown, enter an identifier, etc.). By combining these various elements and making them interact with each other, it is possible to create very complex user interfaces. [See an example]

[Layout image]

Add visu elements and change its type​

Clicking on the [Add icon] an element is added in the table. By default, the type of this element is Display, but it can be changed clicking on the selector.

Adjust the position and size of the elements​

To set the size and the position of an element, it can be done updating the values of the following columns:

  • Top: sets the vertical position. Default value: 0px.
  • Left: sets the horitzontal position. Default value: 0px.
  • Width: this property sets the element's width. Default value: 100%.
  • Height: this property sets the element's height. Default value: 100%.
caution

All these properties only accept values in pixels (px) or percentages (%). It is also important to note that if you want the elements to have a responsive behavior, the values must be in %.

Adjust the visibility of the elements​

  • Z index: sets the z-order of an element. In case the elements have overlapping, the element with a bigger z-index will be rendered on top. Default value: 1;
  • Hidden: if true, the element exists but is not displayed. Default value: false;
caution

When the runtime is stopped, the last value of these properties is the one that persists. Stopping the runtime does not reset the values to these ones in the table.

Extra configuration of some elements​

There are some elements that have an extra-static configuration. Is the case of the Time graph elements. To configure it click on the [Settings icon] button and a popover will be opened.

Visu elements​

Display​

Display elements are used to continuously show a changing image in production. By double-clicking on these elements, you can enter display-2d mode to view the sent image in a larger format.

Time graph​

Time graph elements are designed to display a temporal graph, where the x-axis updates at a configurable interval while showing various types of data (points, lines, bars, etc.). These data are dynamically added in production.

Text box​

The objective of text box elements is to display text that can change dynamically in production. In addition to the text content itself, attributes such as font size, color, alignment, etc., can also be adjusted.

Static image​

Unlike Display-type elements, Static Image elements are designed to contain static images. This would include cases like logos that are intended to remain consistent and unchanged.

Table​

Table elements are used to display data in a table format. Both the content and the background color of the cells, as well as the number of rows and columns, can dynamically change. Additionally, these tables can be interactive, allowing users to select one or multiple rows.

Push button​

The push button is an element that responds to user clicks in production. It dynamically changes text, color, size, and enables or disables its functionality. It is useful for immediate actions such as starting a process, submitting data, or activating features.

Toggle button​

The toggle button is an interactive element that allows users to toggle between two states, typically "on" and "off". In production, when the user presses the switch, it visually updates to reflect the current state and can trigger actions associated with that state. Its value can also be updated programatically, without the action of a user.

Text input​

A text input is an interactive element that allows users to input and edit text in production. As users type, the content of the text input can be dynamically captured for processing or data storage purposes.

Selector​

A text selector is an element that allows users to select a predefined text option from a dropdown list. It is useful for quickly choosing from available options and can dynamically capture the user's selection for processing or data storage purposes.
