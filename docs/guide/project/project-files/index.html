Project files

Projects are the main way to organize different application scenarios within OneVision. A project is a saved collection of cameras, Deep Learning brains and runtime sequences. You can create as many projects as you want, and switch between them at any time.

Most of the time, you will be working with a single project per application. But in cases of multiple product references that cannot be handled by a single project (the most desirable scenario), you can create multiple projects and switch between them manually or with a communication signal from a PLC or robot.

tip

Important: Communication interfaces are not part of the project file because they are a static configuration of the application (the same for all projects).

The project menu​

The Project menu is located in the topbar. It contains all the commands that you can use to manage projects.

New project​

Create an empty project, as explained in the Quickstart section.

Open project​

Opens the Open project popup, explained below, where you can select a project to open.

Close project​

Closes the current project. If you have made changes to the project, you will be asked to save them before closing. The current project is shown in the topbar.

Open recent​

Shortcut to open the most recently used project.

Save project​

Saves the current project if it has changes. This command is also available in the left bar inside the Project section.

Export project​

Saves the current project to a file. To later use the exported project, the import mechanism is used, as explained below.

Fullscreen​

Toggles the application to fullscreen mode.

Exit​

Stops the application. If you have made changes to the project, you will be asked to save them before closing.

tip

Important: This is the only way to exit the application. The X button on the right in the topbar only hides the application, but it does not stop it.

Open project popup​

Project files are managed internally by the application, they cannot be opened directly from the file system as you would do with a document of a text processor or a spreadsheet.

All the project files that you have saved are listed in the Open project window, which you can open from the Project menu. All projects are identified by a unique name and a Last edited date. You can also view some other details about the project in the right-side panel, as the author and the version of OneVision that was used to create the project.

To open a project, select it from the list and click Open, or double-click on it.

Clone project​

To clone one of the existing projects, click on the settings button next to the project and select Clone. It will require you to enter a new name for the project.

Import a project​

To import a project, click on the Import button in the bottom of the popup. It will open a file selection dialog where you can select the project file to import. The imported project will be added to the list of projects.

Retargeting​

When the project that is imported has been created with an older version of OneVision, before importing the project you will be asked to retarget it to the current version. This is necessary because the project file format may have changed between versions, and the application needs to know how to convert the old format to the new one.

In case of very old versions of the software it might not be possible to retarget the project. In this case, you will need to open the project with the version of OneVision that was used to create it, and recover the information manually, for example copying the program sequences.
