Programming

The work in OneVision is done by creating runtime sequences. These sequences are written in OneVision Scripts, a scripting language that is based on Typescript.

In the Programming section, there are 4 sub-sections that can be selected from the upper horizontal menu:

  • Sequences: the list of sequences in the project.
  • Globals: the list of global variables in the project (not editable).
  • Types: a list of custom types to be used in the code.
  • Actions: a list of runtime variables that can be modified during runtime via a side-panel in the Home view.
info

We do not cover here the specifics of the programming language itself. For information about the language and API, please refer to Programming reference.

Add a sequence​

To add a sequence, click on the button in the top left corner.

Sequence name​

The name of the sequence. Must be unique and is stored in the project-file. It can be changed later.

tip

When there are many program sequences, giving them intuitive names really helps to identify them.

Sequence type​

There are four types of sequences:

TypeDescription
INITThese sequences execute at the beginning.
ENDThese sequences execute at the end.
SIMPLEThese sequences execute after INIT and before END, with a definite start and end.
CYCLICThese sequences execute in a cyclical manner

Sequence status​

Once the sequence is added, it will appear a new tab in the top bar with the name of the sequence. You can delete a sequence any time by clicking on the delete button in the top right corner of the sequence tab.

  • On the left, the type of the sequence is displayed.
  • In the middle, the state of the sequence is shown. If the user adds a sequence but doesn't click on Save changes, the status will be Not loaded. Once saved, the state will be Ready. [TODO: Specify all possible states]
  • On the right, there is a summary of the sequence, including the Name, Type, Enabled status, State, and Time of execution (which will be empty if not yet executed).

Sequence options​

Clicking the Options button will display the Sequence control popup. Here the user can edit the sequence name, enable/disable the sequence or change its type.

info

Disabling a sequence is useful when we don't want it to execute when running the program, but we also don't want to delete it.

Debugging​

Debug mode in OneVision is a vital tool for testing and debugging programming sequences. This mode allows executing sequences step by step, observing real-time results, and efficiently correcting errors.

Activate Debug mode​

To activate the debug mode, click the Debug button in the left side menu, under Discard changes.

Using debug mode​

Once activated, debug mode allows you to:

  • Step through sequences: Enables observing the result of each individual instruction.
  • Inspect variables and states: Verify variable values and program state at each step.

Use the debug toolbar on the right to:

  • [Start icon] Start program
  • [Jump icon] Jump to the next breakpoint
  • [Restart icon] Restart
  • [Stop icon] Stop execution

Breakpoints​

Breakpoints are breakpoints in the code where execution will automatically stop to allow inspection of the current state of the program.

Add Breakpoints​

  1. Open the sequence you want to debug.
  2. Click on the line of code where you want to add the breakpoint.
  3. Select Add breakpoint from the context menu.

Delete Breakpoints​

  1. Click on the existing breakpoint.
  2. Select Remove breakpoint from the context menu.

Errors ocurred during execution​

When running debug mode:

  1. Start Debugging: Click the start button to start debugging step by step.
  2. Observe the Results: Each step will display the results in real time, allowing you to verify the expected behavior of the sequence.
  3. Inspect State and Variables: Use the interface to view the state of variables and other parameters at each breakpoint.

Error Management​

While debugging, errors will be automatically highlighted, providing detailed messages to identify and fix problems.

Error Handling​

  1. Error Identification: When an error occurs, the execution will stop and a detailed message will be displayed on the interface.
  2. Diagnosis: Review the error message for specific information about the cause.
  3. Fix: Modify the code as necessary and resume debugging.
