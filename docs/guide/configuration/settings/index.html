Settings

The Settings tab is a miscellaneous collection of settings that configure the behavior of the application. There are two sub-sections that can be selected from the upper horizontal menu:

  • General: Configures various parameters divided by services.
  • Users: Manages user accounts and permissions within the application.

Brains​

List of parameters that define the configuration of the Brain service. [See Brains]

Log level​

The Log level parameter allows you to specify the verbosity of logging messages for the Brain service. It acts as a filter for the types of messages that are displayed or saved. Adjusting this parameter helps in monitoring and diagnosing issues related to brain operations.

See more about application logs

Max. performance GPU​

If enabled, the brain server will automatically configure the GPU for maximum perfomance and low lattency. Requires administrator privileges. Default value: Enable.

Max. stored profiles​

Maximum number of stored inference profiles per brain. Default value: 10;

Cameras​

Configuration of the Cameras service.

Log level​

The Log level parameter configures the level of detail for logging messages related to the Cameras service. By selecting an appropriate log level, you control the amount of information displayed or saved, which aids in troubleshooting and maintaining camera functionalities.

See more about application logs

Controller plugins​

Define the full path to a DLL or SO file containing a camera controller plugin. More than one plugin can be configured. To add a plugin, click on the button and paste the path in the text input. To remove it, click on the button.

Communications​

Configuration of the Communications service [See Communications]

Log level​

The Log level parameter adjusts the detail level of logging messages for the Communications service. It is essential for monitoring communication activities and diagnosing issues related to data transmission.

See more about application logs

Controller plugins​

Define the full path to a DLL or SO file containing a communication controller plugin. More than one plugin can be configured. To add a plugin, click on the button and paste the path in the input. To remove it, click on the button.

Graphical interface​

List of parameters that define the configuration of Graphical User Interface (GUI).

Log level​

The Log level parameter in the Graphical User Interface (GUI) settings filters the types of logging messages shown. It is useful for adjusting the amount of information displayed based on your debugging or operational needs.

See more about application logs

Auto Update Interval​

This parameter sets how often, in milliseconds (ms), a data table refreshes its content with updated data when the user has clicked the Auto update button. This interval determines how frequently the application retrieves new data from its source and updates the table in the Mes section [See Mes]. Adjusting this parameter balances the need for real-time updates with system performance considerations. Default value 60000;

Open in fullscreen​

This parameter determines whether the application starts in fullscreen mode upon launch. This setting controls the initial display state of the application window. Users can further enter or exit fullscreen mode during use by pressing the F11 key, providing flexibility to adjust the screen layout based on their preference and workflow. Default value: false.

Web Server​

If Enabled, allows remote access to the application. Only one connection can use the application at a time. Users can request control of the application when accessing remotely. Default value: Enable.

Web Server Port​

This parameter defines the network port where the server hosting the application is started. This port number specifies the endpoint through which clients can connect to and communicate with the application over the network. Default value: 5001.

caution

If the Web Server parameter is disabled, the Web Server Port selector will also be disabled.

Enable Users​

It determines whether internal users of OneVision are required to log in to navigate through the application. When enabled, users must authenticate themselves to access the application, enhancing security and ensuring user-specific settings and permissions. If disabled, the application is accessible without requiring a login, allowing unrestricted access to all users. Default value: false. TODO: See Users.

Session time (min)​

The Session Time parameter determines the duration, in minutes, after which an inactive user will be automatically logged out from the application. This setting enhances security by reducing the risk of unauthorized access due to prolonged periods of inactivity. Default value: 10.

caution

If the Enable Users parameter is disabled, the Session Time parameter will also be disabled.

Force Fullscreen​

The Force Fullscreen parameter, when enabled, prevents users from navigating outside of the application by forcing it into fullscreen mode if the configured user level (see Force Fullscreen Level) is not logged in. This setting ensures a focused user experience within the application interface. Default value: false.

caution

If the Enable Users parameter is disabled, the Force Fullscreen parameter will also be disabled.

Force Fullscreen Level​

The Force Fullscreen Level parameter determines the user privilege level at which the application is forced into fullscreen mode if Force Fullscreen is enabled. Users below this privilege level will have the application forced into fullscreen mode to prevent navigation outside of the application interface. Default value: production.

caution

If the Force Fullscreen parameter is disabled, the Force Fullscreen Level setting will have no effect on the application's fullscreen behavior.

Mes​

Configuration of the Mes service [See Mes].

Log level​

The Log level parameter determines the verbosity of logs generated by the MES service. It helps in tracking and troubleshooting activities related to data saving, traceability, and product inspection within MES.

See more about application logs

Enable/Disable MES​

The Enable/Disable MES parameter controls the activation of the MES service. When enabled, the MES service is operational, allowing data to be saved and providing traceability for inspected products. In highly demanding scenarios where MES functionality is not required or needs to be temporarily suspended, disabling this parameter prevents data from being saved. Default value: Enable.

caution

If the Enable/Disable MES parameter is disabled, all the following parameters (Units Buffer Size, Folder path, Max. time range, Max data size and Max number of units) will also be disabled.

Units Buffer Size​

The Units Buffer Size parameter specifies the size of the queue for units waiting to be saved in the database within MES. This queue manages the flow of data for traceability of inspected products. Default value 30

Folder Path​

The Folder Path parameter specifies the absolute path to the folder where all the data of the MES service will be stored. Inside this folder, there will be subfolders created for storing images and image data (Data folder) and the SQLite database (Database folder). By default, this is set to C:/OneVision/Mes.

Max. time range (days)​

The Max time range (days) parameter specifies the maximum number of days that information about a unit (See [results and captures]) will be retained within the MES service. Units older than this specified time range will begin to be deleted from the system. Default value: 30.

Max data size (MB)​

The Max data size (MB) parameter specifies the maximum allowed size for capture data, including images and image data files, within the MES service. If the total size of the data exceeds this specified value, the oldest data will begin to be deleted to maintain storage limits. Default value: 5000.

Max. number of units​

The Max number of units parameter specifies the maximum number of units (entries in the database) that can be stored within the MES service. If the number of units exceeds this specified value, the oldest units and their related data will begin to be deleted to maintain database size limits. Default value: 10000.

info

In some applications, the size of image data stored may not be heavy, but the data stored inside the database can accumulate rapidly. The Max number of units parameter provides a way to control database size and manage storage resources effectively.

Project​

Configuration of the project service.

Log level​

The Log level parameter controls the detail level of logging messages specific to the Project service. Adjusting this parameter allows you to monitor runtime operations and detect issues related to project management efficiently.

See more about application logs

Auto Start Runtime​

The Auto Start Runtime parameter, when enabled, automatically starts the runtime when a project is loaded within the Project service. This can be useful in scenarios where the PC restarts, as it eliminates the need for a user to manually click on the start runtime, allowing the process to start automatically. Default value: Disable.

Enable default project​

The Enable default project parameter, when enabled, automatically loads a specified project when the application is started. Default value: Disable.

Default project​

The Default project parameter specifies which project will be automatically loaded when the application is started. This is a selector where the user can choose from all the projects that have been created in the application. Default value is None.

caution

If the Enable default project parameter is disabled, the Default project will be disabled too.

Runtime​

Configuration of the Runtime service.

Log level​

The Log level parameter in the Runtime service configures the amount of detail in logging messages. This setting is crucial for monitoring runtime activities and diagnosing issues related to the execution environment.

See more about application logs

Signals​

Configuration of the Signals service.

Log level​

The Log level parameter adjusts the verbosity of logging messages for the Signals service. It is essential for monitoring signal processing and operational statuses within the application.

See more about application logs

Heartbeat period (ms)​

The Heartbeat period (ms) parameter specifies the interval in milliseconds at which the heartbeat signal is sent to indicate the application's operational status. Default value is 1000.

System​

Configuration of the system service.

Log level​

The Log level parameter in the System service controls the detail level of logging messages. It is used to monitor system-wide operations and diagnose issues affecting overall application performance.

See more about application logs

Launch on startup​

The Launch on startup parameter, when enabled, ensures that the application automatically starts when the operating system is started. This allows the application to be ready for use without manual intervention each time the system boots up. Default value is Disable.

Launch on background​

The Launch on background parameter, when enabled, allows the application to start and run in the background without displaying its user interface. Default value is Disable.

info

This can be useful for applications that need to perform tasks discreetly or without user interaction upon startup. Clicking on the tray icon in the taskbar will open the application window.

caution

If the Launch on startup parameter is disabled, the Launch on background parameter will also be disabled.

Station name​

The Station name parameter specifies the name of the current station, which will be displayed in the application's title bar. Use a friendly descriptor like Vision line 7 or Robot 3. Default value: Vision.

Users​

The Users tab in OneVision manages internal user accounts intended for generic roles such as production and engineering. These users are not intended for personal use but rather for operational and administrative purposes within the application.

User Levels​

  • Non-Logged In User:
    • Can navigate only in the Home and MES pages.
    • In the Home page, cannot start/stop the program, open/close a project, or exit the application. Will be prompted to log in for these actions.
    • Cannot create new projects, export projects, or perform other project-related actions.
  • Production User:
    • Can navigate in the Home and MES sections.
    • Can start/stop the program, open/close projects, and exit the application.
    • Cannot access the Edition or Config sections.
  • Engineering User:
    • Can perform all actions and navigate across all sections.
    • This user level is designated for users who can edit projects, including changing camera features, editing sequences, and loading new brains.

Change User Password​

To change the user password:

  1. Click on the key icon button in the users table. This will open a popup window to set the new password.
  2. Enter the desired new password in the provided field within the popup.
  3. Click on the Save button within the popup window to confirm the new password.
  4. Navigate to the left sidebar and click on Save changes under the Settings tab.
  5. The application will restart automatically with the new password set.

Limited User Management

This predefined generic user accounts are provided for operational roles such as production and engineering. These users cannot be added, removed, or have their privileges edited. They are designed for specific functions within the application.

Activate Users and Log In​

To activate users:

  1. Navigate to the Settings section under the General tab.
  2. Find the Graphical interface settings and set the Enable Users parameter to Enable.
  3. Save changes in the configuration.

After saving, the application will automatically restart. Once restarted, a Sign in button will appear on the blue top bar next to the application buttons (See appbar).

Click on the Sign in button to open the login popup.

For initial login, use the default passwords provided below:

Application Logs​

Application logs are essential for tracking and diagnosing operational events and issues within OneVision. These logs provide valuable insights into system behavior, errors, and performance metrics, aiding in troubleshooting and maintenance tasks.

Types of logs​

Log TypeDescription
INFOProvides informational messages about the application's operational status and significant events.
DEBUGIncludes detailed debugging information useful for troubleshooting and diagnosing issues.
WARNLogs warnings and potential issues that do not prevent the application from functioning but may require attention.
ERRORLogs critical errors and exceptions that impact the application's functionality or user experience.

Log level parameter​

Each service in OneVision includes a Log Level parameter that acts as a filter for the type and detail of logs displayed or saved. Selecting a specific log level determines which categories of logs are visible or recorded.

Log level filter​

  • INFO (Production): Displays informational, warning, and error messages relevant to monitoring the application's production environment.
  • DEBUG (Show All Logs): Shows all logs, including detailed debugging information useful for diagnosing issues during development or testing phases.
  • WARN (Just the Warnings and Errors): Filters logs to display warnings and errors, providing a balance between information and critical alerts.
  • ERROR (Just the Errors): Shows only critical errors and exceptions, useful for identifying and addressing major issues affecting application functionality.

Adjust the Log Level parameter according to your troubleshooting needs and environment requirements. Choose a level that provides the necessary information without overwhelming the log output.

Show Logs​

To view the OneVision logs:

  1. Right-click on the OneVision tray icon.
  2. Select Show logs from the context menu.
  3. An Explorer window will open at the following location: C:\Users\user\AppData\Roaming\onevision\logs.

Log File Format​

The OneVision logs consist of multiple files named according to their creation date, such as log-2024-06-19-16.log. Each log file contains entries formatted as follows:

15:05:35 | info | ProjectService::loadProject() | Project with name 'example' opened in station 'Vision'

  • Timestamp: Indicates the time when the log entry was recorded.
  • Log Level: Indicates the severity or type of log message (e.g., info, warn, error).
  • Message: Describes the event or action captured by the log.

These logs provide valuable insights into OneVision's operational status, errors, and significant events, aiding in troubleshooting and monitoring system performance.
