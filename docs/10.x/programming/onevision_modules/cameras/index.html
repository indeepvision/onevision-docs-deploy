Cameras

This module is available in the language via de module Cameras, and provides access to the cameras defined in the project by their name (see Friendly name).

Let's suppose you have configured camera_1 in your project, inside the camera there are member functions that will be suggested to you when you write Camera.camera_1.

startAcquisition​

  • OneVision
startAcquisition(): void;

Description​

Starts the streaming from the camera.

Tips​

  • If the camera is configured to work with software trigger, after calling this function, you will need to call softwareTrigger to acquire an image.
  • If the camera is configured to acquire images continuously, this function will start the acquisition and the camera will start sending images to the application.

stopAcquisition​

  • OneVision
stopAcquisition(): void;

Description​

Stops the streaming from the camera.

Tips​

  • When the program stops, stopAcquisition is called automatically.

softwareTrigger​

  • OneVision
softwareTrigger(): void;

Description​

Stops the streaming from the camera.

Tips​

  • When the program stops, stopAcquisition is called automatically.

waitTriggerReady​

  • OneVision
softwareTrigger(): void;

Description​

Stops the streaming from the camera.

Tips​

  • When the program stops, stopAcquisition is called automatically.

getImage​

  • OneVision
softwareTrigger(): void;

Description​

Stops the streaming from the camera.

Tips​

  • When the program stops, stopAcquisition is called automatically.

waitImage​

  • OneVision
function waitImage(timeout?: number): boolean;

timeout (optional): Timeout in milliseconds. If not specified, the function will wait indefinitely.

Description​

Waits until a new image arrives from the camera, and then returns true. Optionally, you can specify a timeout in milliseconds, after which the function will return false if no image has arrived.

Tips​

  • When stopAcquisition, the pending wait will be cancelled.
  • You can cancel the wait by calling cancelWaitImage.

cancelWaitImage​

writeBoolean​

  • OneVision
function writeBoolean(featureName: string, value: boolean): void;

featureName: The exact name of the feature as it appears around brackets in the feature list of the camera (see Camera features).
value: The value to write.

Description​

This function will try to find the feature with the specified name in the camera, and write the specified value to it. If the feature is not found, an exception will be thrown.

Tips​

  • If the camera is acquiring images, or in video mode, features might be locked and this function will throw.
