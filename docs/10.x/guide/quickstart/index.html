Quickstart

Let's discover OneVision in less than 5 minutes.

Layout of the application​

When you open OneVision you will see the following layout. There are 4 main sections in the application:

Home​

This is the production view, where the program results are shown, and also where you can manually trigger cameras and execute inference on AI models. This is what the application will show most of the time, as during production it is the most useful. Read more.

MES​

This is an internal database where production images and results are stored. The information stored here can be queried for result analysis by vision engineers or quality personnel. The size of the database is automatically managed, removing oldest entries. Read more.

Project​

In the project section you can define a collection of cameras, brains, runtime sequences and a visualization, specific to a particular use-case. Different project files can be stored and loaded when the product reference changes, or to experiment with different configurations. Read more.

Configuration​

The configuration determines the behavior of the application as a vision workstation within a factory. As such, it is static, different configurations cannot be loaded like a project. It includes the configuration of communication interfaces with the outside world, the configuration of the system signals, and the general settings of the software. Read more.

Application bar​

This is the top blue bar of the application, which is always visible. It contains the following elements:

  • Project: this is where you can create a new project, open an existing project, save the current project, etc.
  • Help: this is where you can find the documentation, license manager, etc.
  • Current opened project: this is where you can see the name of the current opened project.
  • Application buttons: standard buttons to minimize, maximize and close the application. The close button only hides the application, it does not close it.
tip

To close the application you must use the Exit button in the Project menu in the topbar.

Create a new project​

We will start by creating a new project. Click on the Project menu in the top bar and then click on New Project.

A popup will show where you can enter the name of your project (for example quickstart_ov), author (you or your company) and a description (optional).

Once you have created the project, the name of the project will appear in the top bar.

Go to the Project section in the left menu, you will see we have created an empty project.

Now let's start adding some elements to the project.

Add an emulation camera​

From the Project section, click on Cameras. Here is where all the cameras used by the application must be added. Each camera will appear as a separate tab.

To add an Emulation camera click on the add button in the top left corner.

A popup will appear where you can enter the name of the camera (for example emulation_cam),

Now you can try to save the changes clicking on Save in the left menu. This will save the current changes to the project file. Click Discard changes if you want to discard the changes made to the project.

After saving, the emulation camera status will change from Not initialized to Configuration error. This is because we have not yet configured the camera, primarily we have not introduced a serial number.

Because emulation cameras are an internal feature of OneVision, the developers have defined 8 possible serial number for emulation cameras. For the moment we will use the first emulation camera, so enter EMU1 in the Camera serial field on the Device selection section.

Now click on Save in the left menu, the emulation camera status will change to Connected.

Trigger the camera​

If you go to the Home section you will be able to manually trigger the camera and see the images captured by it.

If you click on Single shot you will force the capture of an image by the camera. By default, the images generated by Emulation cameras are 1920 x 1080 monochrome images, with a moving diagonal pattern.

Buttons for manual camera operation are:​

  • Single shot: Force the capture of a single image by the camera.
  • Start video: Force the start of a video stream from the camera (normally limited to 15 fps).
  • Start acquisition: Start the normal acquisition of images from the camera.
  • Software trigger: Only enabled if the camera is in acquisition mode. Sends a software trigger to the the camera if it supports it.
  • Switch camera: Switch between different cameras in the project (we have for the moment a single camera).

Add a brain​

The AI models or brains allow us to execute analysis based on Artificial Intelligence. Let's add a new brain to the project.

From the Project section, click on Brains. Here is where all the brains used by the application must be added. Each brain will appear as a separate tab.

To add a Brain click on the add button in the top left corner.

A popup will appear where you can enter the name of the brain (for example test_brain). Once added, two different screens will show for the brain, the Model selection screen is where you can select the AI model to use for the brain, and the Configuration screen is where you can configure different parameters for the brain, such as if inference will run on GPU or CPU.

We need to import an AI model before assigning it to the brain. From the Model selection screen, click on the add button in the bottom of the empty table. Select a model file from your computer and click on Open. The imported file must have the extension .brain.

In the example we have added a model called dry_fruit_sample with version number 2.

Now select assign the model to the brain by clicking on the Select button. Save the changes made to the project and you will see how the brain status changes from Not initialized to Ready.

If no model is selected, the brain status will be Initialization error.

Execute inference on a brain​

Let's move back to the Home section and see how the brain works. Click on the Single shot button to capture an image from the emulation camera. The brains now will be enabled, click on Inference to execute a manual inference on the image just captured.

Because the brain is not supposed to work on the image captured by the camera, the inference result will be a very low probability segmentation. You can see the objects added by the inference on the Objects right-side panel. In this case we have 5 results: almonds, hazlenuts, peanuts, pistachios and raisins.

We will now open an image from disk with dry fruits. Click on Open image and select the image dry_fruits_001.jpg. Now execute inference again and you will see how the results are much better.

Create a program sequence​

Let's start writing some code. We want to create a program that opens images from a folder and executes our brain on them. To do this we will create a program sequence.

Go to the Project section and then click on Programming. In the Sequences screen, click on the add button in the top left corner. This will open a new sequence popup, where you can enter the name of the sequence (for example test_sequence) and also select the type of sequence, in this case we will select SIMPLE.

tip

Sequence types

  • SIMPLE: It is executed once when the program starts and then it is finished.
  • CYCLIC: It is executed continuously, once finished it is executed again.
  • INIT: It is executed once when the program starts, before any other sequence execute.
  • END: It is executed once when the program is stopped, after other sequences have ended.

Now add the following lines of code:

Quickstart runtime sequence

// OneVision quickstart program

// Configure location of the images
const var IMAGES_FOLDER = "C:\dry_fruits_images\";

// List the images in the folder
var images_list = File.listImagesFromDir(IMAGES_FOLDER);

for (var i = 0; i < images_list.length; i++) {
    // Create the complete path to the image
    var image_filepath = IMAGES_FOLDER + images_list[i];

    // Load the image into an Image vision element
    var loaded_image = Image.fromEncodedFile(image_filepath);

    // Resize the image to match the input size
    loaded_image = loaded_image.resize(1224, 1024);

    // Execute inference on the image
    var inference_result = Brains.test_brain.inference(loaded_image);

    // Change the color of the segmentations
    inference_result.almonds.setColor("#ff0000");
    inference_result.hazelnuts.setColor("#00ff00");
    inference_result.peanuts.setColor("#0000ff");
    inference_result.pistachios.setColor("#00ffff");
    inference_result.raisins.setColor("#ff00ff");

    // Create a capture to send to the GUI
    var capture = new Capture("Dry fruits", loaded_image);
    capture.addVisionElement("almonds", inference_result.almonds);
    capture.addVisionElement("hazelnuts", inference_result.hazelnuts);
    capture.addVisionElement("peanuts", inference_result.peanuts);
    capture.addVisionElement("pistachios", inference_result.pistachios);
    capture.addVisionElement("raisins", inference_result.raisins);

    // Send the capture
    Visu.display.sendCapture(capture);
    Visu.update();

    // Wait some time
    waitTime(1000);
}

This code will open the images in the configured folder and execute inference on them.

Save the changes made to the project by clicking on Save in the left menu.

Run the program​

The program (or runtime) is executed in the Home section. Click on Start program to start the program. You will see how the images are opened and inference is executed on them. For each image a new capture is shown in the GUI.

After all the images have been processed and the for loop is finished, the program will stop automatically.

You can also stop the program manually by clicking on Stop program.
